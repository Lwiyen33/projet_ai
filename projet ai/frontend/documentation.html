Problème métier · Données · Modèle · Intégration web

1. Problème métier

Aider à décider combien commander en s’appuyant sur le stock, la demande (ventes) et le temps de livraison (nombre de jours entre la commande et la réception), plutôt qu’une règle fixe.

2. Données (Pandas, visualisation Matplotlib)

Le fichier data/stock_synthetic.csv est généré par scripts/generate_data.py. Le notebook notebooks/analyse.ipynb présente nettoyage, graphiques et entraînement.

  • Variables : catégorie, stock, ventes/semaine, jours de livraison, mois
  • Cible : quantité recommandée (recommended_order_qty)

Données synthétiques — à mentionner dans le rapport.

3. Modèle (scikit-learn)

Random Forest dans un pipeline (prétraitement + régression). Entraînement : python scripts/train_model.py → models/stock_model.joblib.

4. Intégration web

Le backend Flask gère une liste de produits (fichier data/products.json) via GET/POST /api/products, PATCH/DELETE /api/products/<id>, et la suggestion POST /api/products/<id>/suggest. Le simulateur utilise POST /api/predict. L’interface est en HTML/CSS/JS (fetch).
